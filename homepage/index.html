Hexo + NexT博客搭建过程记录（三）—— 自定义博客主页

发表于 2022-04-10 更新于 2022-04-20

前言

NexT主题默认的主页是我们的文章列表。如果我们想要自定义一个主页显示我们的个人信息和前往其他平台的链接，该怎么做呢？

在浏览器上看到的我们的博客实质上都是渲染后的html文件，那么自定义一个html文件作为我们的博客主页应当是可行的，本文出于这一思路实现自定义主页的目标。

添加自定义主页文件

首先我们当然要添加自定义主页的文件。据我测试，只要在source目录下创建名为index.md的文件，该文件就会以文章的形式作为我们的主页。

当然了，index.md是被渲染为index.html后再在浏览器中显示的（在执行hexo generate命令后就可以在public目录下看到相应的文件）。因此我们直接在source目录下创建index.html文件即可。

我们可以对index.html文件进行编辑，会发现我们的主页还是以博客文章的形式出现的。因此我们需要在文件的front-matter部分选择关闭渲染：

关闭渲染后我们的主页就会以正常网页的形式显示了。擅长网页开发的大佬们也已经可以自由发挥了。（作为菜鸟的我当然选择抄大佬们的作业~(￣▽￣)~）

我选择的主页模板效果如下：

如上所述，我们把主页模板的文件放到source目录下。问题在于直接把一堆文件丢在这里会使得source文件夹内十分杂乱，于是我把除index.html文件外的其他网页文件都移入了新建的homepage文件夹内（主页文件必须在source目录下，否则无法识别），随后简单修改了下html文件内的引用路径。

主页生成位置更改

站点配置文件下有一个index_generator的配置项。
我们需要把其下的path改为之前创建的homepage文件夹，否则在生成博客时还是会生成默认的文章列表文件顶替掉我们的主页文件。
（实际上path只要不是默认值''就行，但是在生成博客文件时会根据这里的path创建文件夹，我们把path设置为已有的homepage是为了生成文件里少一个文件夹。）

最终效果

点击Notes即可进入博客，我把Notes指向了博客的归档页面。

在博客页面点击Home，就回到了我们定义的主页面。

Hexo + NexT博客搭建过程记录（一）—— Hexo安装

发表于 2022-04-04 更新于 2022-04-09

所需工具

Node.js

Node.js是一个基于Chrome V8引擎的JavaScript运行环境，让JavaScript 运行在服务端的开发平台，它让JavaScript成为与PHP、Python、Perl、Ruby等服务端语言平起平坐的脚本语言。

Git

Git是一个开源的分布式版本控制系统，可以有效、高速地处理从很小到非常大的项目版本管理。

Hexo

Hexo是基于Node.js的博客框架，可以方便地生成静态博客页面，并部署在Github上。

搭建过程

安装Node.js

直接到官网下载安装即可，现版本会自带对Node.js依赖的包进行管理的npm。
进入控制台，输入以下命令，如果出现版本号则安装成功

node -v
npm -v

如果想要调整npm安装的全局模块以及cache缓存的路径，可以调整环境变量。
分别在系统变量与环境变量的Path下作如下调整（具体路径自选）：

调配完变量后在控制台执行如下命令即可：

npm config set prefix "D:_global"
npm config set cache "D:_cache"

最后全局安装express模块测试一下（所有的安装操作都尽量在管理员权限下执行，否则可能有安装文件缺失或者莫名其妙的报错）：

npm install express -g

配置成功的话会看到express文件安装在刚才配置的全局模块路径的文件夹中。

安装Git

直接到官网下载安装即可。
安装后进入控制台输入命令，出现版本号则说明安装成功。

git --version

在Github搭建仓库

在Github创建仓库，仓库名必须为username.github.io，其中username以点击右上角显示出的名字为准。

生成SSH Keys

进入Git Bash界面，输入指令：

ssh-keygen -t rsa -C "邮件地址"

连点四次回车，此时会在C:\Users\Username的.ssh文件夹下生成文件。

进入GitHub设置里的SSH and GPG Keys页面，选择新建SSH keys。打开.pub文件，拷贝里面的所有内容到SSH Keys，最后Add SSH key即可。 最后输入命令检验是否成功：

ssh -T git@github.com

本地生成博客

选择一个空文件夹作为博客在本地存放的位置，在空文件右键打开Git Bash页面，输入指令安装hexo。

npm install -g hexo

随后进行初始化：

hexo init

如果出现unable to access类的报错，一般为网络问题，多次重试即可。
如果和我一样出现npm命令无法识别等等的报错，可以尝试安装cnpm：

npm install -g cnpm --registry=https://registry.npm.taobao.org

再转用cnpm进行上述操作即可:

cnpm install -g hexo

出现“Start blogging with Hexo”!，就说明我们的博客安装成功了！初始化后就可以在文件夹中看到本地的博客内容了。

Hello World！

发表于 2022-04-04 更新于 2022-04-09

搭建一个属于自己的博客是我长久以来的愿望，然而因为技术不足的原因久久没有实现。原本的计划是等到学校开展Web有关课程的时候再付诸行动，但随着学习的深入，我对搭建博客的需要也愈发迫切。无论生活还是学习中都有很多事要记录，而现在我电脑上的笔记已经堆积如山。在手头没有电脑的情况下访问笔记与文章十分不便且不说，并且还有丢失的风险。

为什么不选择CSDN、博客园等公开的博客平台？首先是因为我预计未来写的内容可能杂七杂八，不适合在这种技术向的平台发表。其次在公开平台发文章不符合我个人不对外输出观点的观念。因此简书、知乎这种功能残缺、内容混杂的平台也不在考虑范围内。总之就是一个强迫症对现有平台左右不如意，与其在这挑三拣四，不如自己搭建一个，这就是本博客的由来。

经过一番检索，孤陋寡闻的我才知道了有Hexo这种好东西。几乎不需要任何的Web功底，又能完美地契合我的需求。于是我在知道Hexo的当天就着手开始搭建，过程虽有坎坷但大体还算顺利。本来在一周前就已经可以访问并上传文章了，不过我又断断续续调整了许久才基本达到了自己想要的效果。接下来先简要记录一下博客的搭建过程（主要是记录踩的坑），以作备忘。
